# **Fragile** - Lightweight File Integrity Monitor

Fragileは、システムの重要なファイルが意図せず書き換えられる「改竄」をリアルタイムで検知するために開発された、Pythonベースの軽量セキュリティツールです。大切なデータの完全性をSHA-256ハッシュアルゴリズムによって保護します。

---

## プロジェクトの概要

Fragileは、事前に指定されたファイル群のハッシュ値を計算し、メモリ上に保持します。その後、1秒間隔のループによって各ファイルをスキャンし、元の指紋と現在の指紋に差異が生じた場合に警告を発します。このツールは、単なる変更検知に留まらず、セキュリティエンジニアリングにおける"Integrity"の概念を具体化したものです。

---

## 技術的な特徴と仕組み

プログラムの根幹を支える技術要素を以下の表にまとめました。

| 要素 | 技術的詳細 | 期待される効果 |
| :--- | :--- | :--- |
| **計算アルゴリズム** | SHA-256 (Secure Hash Algorithm 2) | 高い衝突耐性と安全性の確保 |
| **メモリ管理** | 4096バイト単位のチャンク読み込み | 数GBを超える巨大なファイルでも安定動作 |
| **クロスプラットフォーム** | platformモジュールによるOS判定 | Windows/Linux/Mac環境を問わない汎用性 |
| **状態管理** | 動的な辞書型（dict）によるスナップショット | 監視中のエラーにも柔軟に対応 |



内部処理においては、`sha256_sum`関数がファイルオブジェクトをバイナリモードで読み込み、反復処理によってハッシュ値を更新します。これにより、メモリの消費量を一定に保ちながら正確なハッシュ文字列（str型）を生成することが可能です。

---

## 導入と実行方法

本プロジェクトは、高速なPythonパッケージマネージャーである `uv` を活用して構築されています。まずはリポジトリをクローンした後、プロジェクトのルートディレクトリで `uv sync` を実行してください。これにより、`pyproject.toml` と `uv.lock` に基づいた厳密な仮想環境が自動的に構築されます。

監視を開始するには、まず `file_list.txt.example` を `file_list.txt` とリネームし、監視対象としたいファイルのパスを記述してください。その後、`uv run fragile.py` を実行することで、Fragileが起動します。Windows環境では `winsound` による警告音が、それ以外のOSではターミナルのベル通知（ASCIIベル）が異変を即座に知らせます。
